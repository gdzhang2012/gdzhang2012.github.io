https流程分析

2018-05-08

前段时间面试时被问到https连接建立过程，没有回答上来，只笼统地说了两句。其实在以前的工作中也接触过几次https，大多跟安全性问题有关。在解决这些问题时会了解一些https相关的知识，但并不系统，也不全面，并且过一段时间就忘了。所以打算总结一下，做个记录。

https在我理解就是http + ssl。http没有什么连接过程，所以https连接过程也就是ssl的建立过程。ssl连接过程中，client与server协商若干选项，并可相互验证对方的身份，牵扯到对称/非对称加密算法、证书与数字签名等等。这些知识在分析https连接建立过程中都会有所涉及，所以先记录下https连接过程这个主干，以后就可以方便地延伸到各个知识点。

2018-04-24

序言

看了OpenWrt firewall文档后，打算通过TRACE target学习/验证文档中的Packet flow一节。

1.编译TRACE target

make menuconfig时，需要同时选择iptables kernel space和user space模块选项, 如下：

CONFIG_PACKAGE_kmod-ipt-debug=y
CONFIG_PACKAGE_iptables-mod-trace=y

2.获取TRACE输出

首先执行iptables命令，以OUTPUT为例：

root@LEDE:~# iptables -t raw -A OUTPUT -p icmp -j TRACE

接着执行ping命令：

root@LEDE:~# ping -c 1 www.baidu.com

OpenWrt+Raspberry折腾

2018-04-05

序言

近期打算深入学习OpenWrt，恰手边有一Raspberry 3, 于是将Raspberry作为硬件载体。

1.配置Raspberry路由器

为Raspberry添加一USB-to-Ethernet网卡，然后修改OpenWrt网络、防火墙等配置,实现路由器功能。

USB-to-Ethernet网卡信息如下：

usb 1-1: Product: AX88772B
usb 1-1: Manufacturer: ASIX Elec. Corp.

make menuconfig时添加luci, asix网卡支持：

$ ./scripts/diffconfig.sh

CONFIG_TARGET_brcm2708=y
CONFIG_TARGET_brcm2708_bcm2710=y
CONFIG_TARGET_brcm2708_bcm2710_DEVICE_rpi-3=y
CONFIG_DEVEL=y
CONFIG_BUILD_LOG=y
# CONFIG_FEED_luci is not set
# CONFIG_FEED_packages is not set
# CONFIG_FEED_routing is not set
# CONFIG_FEED_telephony is not set
CONFIG_PACKAGE_kmod-libphy=y
CONFIG_PACKAGE_kmod-mii=y
CONFIG_PACKAGE_kmod-usb-net=y
CONFIG_PACKAGE_kmod-usb-net-asix=y
CONFIG_PACKAGE_libiwinfo=y
CONFIG_PACKAGE_libiwinfo-lua=y
CONFIG_PACKAGE_liblua=y
CONFIG_PACKAGE_libubus-lua=y
CONFIG_PACKAGE_libuci-lua=y
CONFIG_PACKAGE_lua=y
CONFIG_PACKAGE_luci=y
CONFIG_PACKAGE_luci-app-firewall=y
CONFIG_PACKAGE_luci-base=y
CONFIG_PACKAGE_luci-lib-ip=y
CONFIG_PACKAGE_luci-lib-jsonc=y
CONFIG_PACKAGE_luci-lib-nixio=y
CONFIG_PACKAGE_luci-mod-admin-full=y
CONFIG_PACKAGE_luci-proto-ipv6=y
CONFIG_PACKAGE_luci-proto-ppp=y
CONFIG_PACKAGE_luci-theme-bootstrap=y
CONFIG_PACKAGE_rpcd=y
CONFIG_PACKAGE_uhttpd=y
CONFIG_PACKAGE_uhttpd-mod-ubus=y

Raspberry启动后，可以看到asix网卡已可正常工作,接口名为eth1：

root@LEDE:/# dmesg | grep asix
[    6.119302] asix 1-1.5:1.0 eth1: register 'asix' at usb-3f980000.usb-1.5, ASIX AX88772B USB 2.0 Ethernet, 00:00:00:00:00:9f
[    6.136141] usbcore: registered new interface driver asix
[    8.867363] asix 1-1.5:1.0 eth1: link down
[   10.790620] asix 1-1.5:1.0 eth1: link up, 100Mbps, full-duplex, lpa 0x4DE1

通过console查看network初始配置：

root@LEDE:/# cat /etc/config/network 

config interface 'loopback'
        option ifname 'lo'
        option proto 'static'
        option ipaddr '127.0.0.1'
        option netmask '255.0.0.0'

config globals 'globals'
        option ula_prefix 'fdca:3f6e:db20::/48'

config interface 'lan'
        option type 'bridge'
        option ifname 'eth0'
        option proto 'static'
        option ipaddr '192.168.1.1'
        option netmask '255.255.255.0'
        option ip6assign '60'

通过修改network配置,将eth1和wlan0桥接，作为逻辑lan,eth0作为逻辑wan:

root@LEDE:/# cat /etc/config/network

config interface 'loopback'
        option ifname 'lo'
        option proto 'static'
        option ipaddr '127.0.0.1'
        option netmask '255.0.0.0'

config interface 'wan'
        option ifname 'eth0'
        option proto 'dhcp'

config interface 'lan'
        option type 'bridge'
        option ifname 'eth1'
        option proto 'static'
        option ipaddr '192.168.100.1'
        option netmask '255.255.255.0'

接着修改wireless配置,修改过后的内容如下：

root@LEDE:/# cat /etc/config/wireless 

config wifi-device 'radio0'
        option type 'mac80211'
        option channel '11'
        option hwmode '11g'
        option path 'platform/soc/3f300000.mmc/mmc_host/mmc1/mmc1:0001/mmc1:0001:1'
        option htmode 'HT20'
        option country '00'

config wifi-iface 'default_radio0'
        option device 'radio0'
        option network 'lan'
        option mode 'ap'
        option ssid 'LEDE'
        option encryption 'psk2'
        option key 'xxxxxxxx'

最后重启network：

root@LEDE:/# /etc/init.d/network restart

经过上面的步骤，Raspberry路由器就可以上网了。理解起来，dhcp、firewall等模块会使用network配置中的逻辑lan、wan端，所以不需特别处理。
